📋 Notes de version

Historique des mises à jour de VirtualTable RPG.

v1.12 Mai 2026 Actuelle
Fonctionnalités
  • Mode Map unique — nouveau mode d'affichage pour jouer en présentiel avec un écran central : les joueurs voient uniquement la fiche et le chat, la carte est projetée sur un écran/TV dédié
  • Écran central dédié — ?mode=display sur la page de jeu pour afficher la carte en plein écran, sans interface, idéal pour un projecteur
  • Bouton Carte — dans le lobby et dans l'interface de jeu pour ouvrir l'écran central en un clic
  • Dessin & annotations — outil ✏ Draw avec traits libres synchronisés, palette de couleurs, épaisseur variable, undo/clear ; persistant en base de données
  • Fiches multi-systèmes — création de personnage adaptée au système de la campagne (Warhammer, CoC, Pathfinder 2e, Shadowrun, Vampire, Cyberpunk, Savage Worlds) ; stats, compétences, races et classes dynamiques
  • 10 nouveaux systèmes — Cats! La Mascarade, Dune, Star Wars, Le Seigneur des Anneaux, Paranoia, Tomorrow City, Starfinder, Shadowrun, Vampire, Cyberpunk Red, Savage Worlds ; avec descriptions détaillées et logos dans le sélecteur
  • Styles de dés 3D — 10 thèmes visuels (Classique, Rubis, Saphir, Émeraude, Améthyste, Doré, Ivoire, Glace, Lave, Nocturne) avec sélecteur de style ; animation réaliste avec chute et rebond
  • Mode Table tactile — écran verrouillé avec QR code pour les tables tactiles horizontales ; idéal pour jouer en présentiel avec figurines
  • Système de ciblage — ciblez un token via clic droit, ligne de visibilité visible par le MJ seul, révélée aux joueurs au lancer de dés si la cible est isolée
  • 4 niveaux de compte — Joueur (👤), Créateur (🛠), MJ Virtuel (🤖), Admin (🛡) avec permissions progressives
  • Page de statistiques — /stats.html avec totaux, dés, critiques, classements joueurs/campagnes, accessible à tous
  • CAPTCHA — protection des formulaires de connexion, inscription et page admin par CAPTCHA mathématique
  • Fonds de carte par thème — les cartes prédéfinies sont triées par pertinence selon le système de campagne
  • Dés 3D réalistes — physique avec gravité, collisions entre dés, multi-dés (2d6 = 2 dés), table de jeu visible
Améliorations
  • Trust proxy — correction du rate limiter derrière Apache proxy
  • Schéma DB — ajout des colonnes failed_attempts, locked_until, last_login, is_admin, tier manquantes
  • Cartes par défaut — 10 images de cartes pré-installées (crypte, donjon, forêt, village, etc.)
  • Déploiement — script deploy.sh sans sudo (groupe docker) pour les mises à jour ; synchronisation automatique vers le DocumentRoot
  • Suppression de campagne — le MJ peut supprimer définitivement une campagne depuis les réglages
  • Numéros sur les faces des dés — d4/d8/d10/d12/d20 affichent leur valeur sur chaque face
  • Admin middleware — vérification DB du statut admin pour les tokens anciens
v1.11 Mai 2026 Stable
Fonctionnalités
  • Montée de niveau D&D 5e — bouton + dans la fiche personnage pour envoyer une demande au MJ ; choix entre moyenne fixe ou jet de dé pour les PV gagnés
  • File d'approbation MJ — panneau dédié (bouton ⬆ Niveaux avec badge) listant toutes les demandes en attente ; le MJ approuve ou refuse avec message optionnel
  • Calculs automatiques D&D 5e — à l'approbation : PV max, bonus de maîtrise et emplacements de sorts (full casters, half casters, Warlock) recalculés selon les tables officielles
  • Synchronisation temps réel — la fiche et la barre latérale du joueur se mettent à jour instantanément sans rechargement
Corrections
  • Tooltip fantôme — la tooltip "Jet de INT ou SAG…" n'apparaît plus au survol de l'indicateur de charge
v1.10 Mai 2026 Stable
Fonctionnalités
  • Descriptions de sorts — 158 sorts avec texte complet (aidedd.org) ; tooltip au survol et panneau détail au clic droit
  • Surlancé (upcast) — clic sur un sort de niveau ≥ 1 ouvre une modale de sélection de l'emplacement ; les dés sont recalculés automatiquement selon la progression indiquée dans la description ("Aux niveaux supérieurs")
  • Sauvegarde dans le chat — le DD de sauvegarde et la caractéristique associée (DEX, CON…) sont affichés dans le fil de chat lors du lancer
v1.9 Mai 2026 Stable
Fonctionnalités
  • Sélecteur d'objets magiques — fenêtre style spell-picker avec 270+ objets D&D 5e (filtre par rareté, type, recherche textuelle)
Sécurité
  • Faille cross-campagne — PUT/DELETE /characters/:charId vérifiait l'appartenance à la campagne mais pas que le personnage en faisait partie ; un membre pouvait modifier des personnages d'autres campagnes
  • Injection via Socket.io — les événements chat_message et dice_roll acceptaient un campaign_id arbitraire ; désormais vérifié contre le campaignId de la session socket
  • XSS dans les handouts — le rendu Markdown via marked.parse() n'était pas sanitisé ; ajout de DOMPurify pour filtrer le HTML dangereux
  • CORS — ALLOWED_ORIGIN pointait sur l'ancien domaine ionos06.cloudfr.net, corrigé vers virtualtable.cloudfr.net
Corrections
  • Sorts non sauvegardés à la création — POST /characters ignorait spells, proficiency_bonus, saving_throws, inventory, subclass et abilities
v1.8 Mai 2026 Stable
Fonctionnalités
  • Dés de sorts configurés — chaque sort utilise ses propres dés selon son type : attaque (1d20 + bonus), jet de sauvegarde (DD annoncé + dégâts), soin (dés + modificateur d'incantation), touche automatique
  • Sélecteurs race / classe / sous-classe — listes prédéfinies D&D 5e et Pathfinder 2e avec saisie libre possible dans la fiche personnage
  • Auto-sélection du personnage actif — le premier personnage est sélectionné automatiquement à la connexion
Corrections
  • Page complètement bloquée — double déclaration const SPELLS_DB entre le script externe et le script inline provoquait un SyntaxError qui empêchait tout le JavaScript de s'exécuter
  • Crash au chargement du personnage — appel parentNode.insertBefore sur un nœud nul dans la section jets rapides
